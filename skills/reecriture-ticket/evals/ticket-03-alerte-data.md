# Éval 03 — Alerte de dépassement data

## Ticket d'origine (à coller tel quel)

> Les clients dépassent leur forfait data sans s'en rendre compte, il faut
> les prévenir.

## Critères binaires

| # | Critère | Oui/Non |
|---|---------|---------|
| 1 | Le canal de notification (SMS, e-mail, push…) n'est pas affirmé : il est marqué `[HYPOTHÈSE]` ou posé en question ouverte. | |
| 2 | Le ou les seuils de déclenchement de l'alerte sont explicités, marqués `[HYPOTHÈSE]` ou à trancher — jamais affirmés comme un fait. | |
| 3 | Les cinq sections fixes sont présentes, dans l'ordre : Contexte, Objectif, Spécifications, Critères d'acceptation testables, Hors scope. | |
