# Catalogue de patterns — Call Builder

> Patterns prescriptifs appliqués dans ce code. Chaque pattern est une règle, pas une
> suggestion. Les nouveaux patterns sont ajoutés en fin de fichier. Chaque pattern
> renvoie vers l'ADR qui l'a ratifié.

## Format d'une entrée

Chaque pattern suit cette structure :

- **Nom** — la clé du catalogue, référencée depuis les revues de code et les ADRs.
- **Quand l'utiliser** — la condition de déclenchement.
- **Description** — la règle prescriptive.
- **Exemple d'implémentation** — pointeur vers le code de référence.
- **Pourquoi ce pattern** — la justification.
- **ADR de référence** — lien vers l'ADR qui l'a ratifié (ou « — » s'il précède le journal).
